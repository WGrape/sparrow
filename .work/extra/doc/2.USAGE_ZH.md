<div align="center"> <h1>使用教程</h1> </div>

## 一、介绍

> 如果访问慢，可以查看[Gitee镜像仓库](https://gitee.com/WGrape/sparrow)。

sparrow是基于Docker一键启动多个服务环境的容器化编排工具。

## 二、环境依赖
本项目完全依赖于Docker环境，所以一定要先安装并启动Docker。

## 三、安装和更新

此项目提供了开发版本和Release版本两种不同的安装方式，二者的区别如下，请根据需求的不同自行选择。

> **稳定性提示**：开发版本是指以 clone 仓库获得的版本（包括 `main` 主分支），会持续更新新功能。虽然每次变更都经过了大量测试，但新功能仍可能带来潜在的不稳定。如遇到相关问题，建议优先下载稳定的 [Release](https://github.com/WGrape/sparrow/releases) 版本包使用。

- 开发版本 ：项目以高频率的速度更新，所以功能和特性一定是最新的！但是无法避免的会出现些bug，不能保证100%稳定
- Release版本 ：项目在某个时间下发布的一个经过多级测试，且稳定运行一段时间后的版本，在[这里查看](https://github.com/WGrape/sparrow/releases)各个Release版本

### 1、开发版本

#### (1) 安装

如果是首次安装，使用如下命令快速完成安装。

```bash
# 获取项目
git clone https://github.com/WGrape/sparrow.git
cd sparrow

# 执行安装脚本
bash _install.sh

# 查看使用帮助
./sparrow --help
```

安装成功后，项目根目录下会自动生成一个 `.env` 配置文件，它是 sparrow 唯一且统一的配置入口，可以查看它的[示例文件](../../.env.template)，在这里你可以修改服务版本、容器命名空间等。

> 1. 如果使用 `./sparrow xxx` 命令时出现语法错误，请尝试使用 bash 运行：`bash ./sparrow xxx`。
> 2. 安装过程中会提示输入 `DOCKERHUB_REPO`，用于配置自己的 sparrow 镜像的远程仓库。默认值是 [docker.io/lvsid](https://hub.docker.com/repositories/lvsid)，如果以后需要修改，直接编辑 `.env` 文件即可。
> 3. 如需帮助，请查看[常见问题](./5.QA_ZH.md)文档。

#### (2) 更新

在安装后，如果需要更新至最新的开发版本，执行以下命令开始更新。

```bash
bash _update.sh
```

### 2、Release版本

#### (1) 安装

如果需要特定版本的Release包，请[点击这里](https://github.com/WGrape/sparrow/releases)下载，然后执行以下操作

```bash
cd sparrow

# 执行安装脚本
bash _install.sh

# 查看使用帮助
./sparrow --help
```

#### (2) 更新

需要注意的是，Release包不支持更新 ！所以在安装完特定版本的Release包后，请不要执行```bash _update.sh```命令。


## 四、快速使用

### 1、启动

使用如下命令启动整个环境中的所有服务。当然这个```所有服务```是可定义的，它由根目录下```/.env```文件中配置的```ENABLE_SERVICE_LIST```数组变量控制。

```bash
./sparrow start
```

#### (1) 启动某一个服务

如果只需要启动某一个服务，使用如下命令即可，传递的```service```就是在```docker-compose.yml```配置文件中的```services```列表中某一个服务的名称，如```phpfpm/nginx/mysql/redis```等

```bash
./sparrow startone {service_name}
```

#### (2) 测试是否启动成功

使用如下命令，会自动在Chrome中打开对应的网页，如果每一个网页都可以成功打开，则说明安装成功

```bash
bash .work/test/run.sh
```

### 2、停止

停止整个环境中的所有服务

```bash
./sparrow stop
```

#### (1) 停止某一个服务

如果只需要停止某一个服务，使用如下命令即可

```bash
./sparrow stopone {service_name}
```

### 3、重启

重启整个环境中的所有服务

```bash
./sparrow restart
```

### 4、工作后台

如果需要通过网页形式管理和查看 sparrow 的服务，使用如下命令启动 web 后台

```bash
./sparrowtool web
```

也可以指定端口号（默认 7979）

```bash
./sparrowtool web 8080
```

## 五、高级使用

### (1) 更换命名空间（CONTAINER_NAMESPACE）

**应用场景：**

在实际应用中，我们可能需要在同一台机器上运行不同环境的同一个服务，切互不干扰。这时，我们就可以使用命名空间功能：

| 命名空间 | 典型用途 |
|----------|----------|
| `test` | 默认本地开发环境 |
| `dev` | 另一套独立的开发环境 |
| `staging` | 同机器上的预发布环境 |

**如何切换环境：**

在根目录下的```./.env```文件中定义了 `CONTAINER_NAMESPACE` 这个关键的隔离变量，它会嵌入到每个容器名称中，容器命名规则为：

```
sparrow_container_{CONTAINER_NAMESPACE}_{service}

# 例如
sparrow_container_test_mysql
sparrow_container_test_redis
sparrow_container_test_nginx
```

所以，只要修改根目录下 `./.env` 文件中的 `CONTAINER_NAMESPACE`，并重新执行```./sparrow startone {service}```命令即可。

## 六、sparrowtool开发工具箱

sparrowtool 除了具有```sparrowtool web```工作后台的功能之外，还提供了镜像上传、新服务创建等高级功能，是 sparrow 的配套工具箱。

### 1、如何上传镜像

#### (1) 上传 Basic 镜像

假设本地有一个服务名为 etcd，对应镜像名为 sparrow-basic-etcd，版本为 3.5.0。如果要上传到 Docker Hub，可执行以下命令：

```bash
./sparrowtool upload -t basic -s etcd -v 3.5.0
```

需要注意：上传后远端镜像版本可能与本地版本不同。

- 本地镜像版本：3.5.0
- 远端镜像版本：etcd.3.5.0_basic.{{timestamp}}

远端版本格式为 `{service}.{version}_{type}.{timestamp}`，可以一眼看出该镜像属于哪个服务、哪个版本、哪种类型，同时时间戳保证历史版本不会被覆盖。

![image](https://github.com/WGrape/lexer/assets/35942268/0c8b2850-940b-4d38-b307-e5508b1df9fe)

为了避免误覆盖远端版本，如果要上传与本地同版本的镜像，必须使用 -r 选项（replace），例如 -r true。

```bash
./sparrowtool upload -t basic -s etcd -v 3.5.0 -r true
```

这样远端会生成一个 3.5.0 版本镜像。

![image](https://github.com/WGrape/lexer/assets/35942268/60b3fc54-2535-4319-83d5-1f7b0dc4a37c)

#### (2) 上传 App 镜像

用法与上面一致。不加 `-r` 时，远端版本同样遵循上述命名规则：

- 本地镜像版本：latest
- 远端镜像版本：etcd.latest_app.{{timestamp}}

```bash
./sparrowtool upload -t app -s etcd -v latest

./sparrowtool upload -t app -s etcd -v latest -r true
```

### 2、如何创建新服务

#### (1) 使用 Sparrowtool 创建

你可以使用 sparrowtool 创建新服务。比如要创建 jupyter 服务，可以执行：

```bash
# 忘记命令时可先查看帮助
./sparrowtool --help

./sparrowtool new -t service -s jupyter -p 2300 -v 0.1.0
```

执行成功后，项目根目录下会新增一个 /jupyter 目录。

##### ① 分配端口号

如果不知道该用哪个端口，可将新服务（例如 langchain）加入 /.work/config/.env.amd64 或 /.work/config/.env.arm64 文件中的 SUPPORT_SERVICE_LIST 变量，然后执行：

```bash
./sparrowtool env -l _port
```

从下图可以看到，系统为 langchain 分配了 [3300, 3400) 端口区间。

<img width="500" alt="image" src="https://github.com/WGrape/sparrow/assets/35942268/ab47592e-2755-4f84-b354-73ef1c6d4b9b">

#### (2) 修改服务配置

你需要在 ./jupyter/.env 中至少修改以下镜像变量：

- IMAGE_OFFICIAL_JUPYTER_NAME：官方镜像名。
- IMAGE_OFFICIAL_JUPYTER_VERSION：可选修改，官方镜像版本，例如 IMAGE_OFFICIAL_JUPYTER_VERSION=18.19.0。
- IMAGE_BASIC_JUPYTER_VERSION：可选修改，basic 镜像版本，建议与 IMAGE_OFFICIAL_JUPYTER_VERSION 一致。
- IMAGE_APP_JUPYTER_VERSION：可选修改，app 镜像版本，建议设为 latest。

以上是必须关注的核心变量。若有更多需求，可继续在 ./jupyter 目录中按需调整。

#### (3) 在 /.work/config/.env 中加入 SUPPORT_SERVICE_LIST 和 ENABLE_SERVICE_LIST

SUPPORT_SERVICE_LIST 和 ENABLE_SERVICE_LIST 定义在 /.work/config/.env.amd64 或 /.work/config/.env.arm64 中。你需要把新服务 jupyter 加进去，例如：

```
SUPPORT_SERVICE_LIST=("xxx" "xxx" "jupyter")
ENABLE_SERVICE_LIST=("xxx" "xxx" "jupyter")
```

#### (4) 删除根目录 /.env

之后建议删除项目根目录下的 /.env 文件，因为系统会自动重新生成新配置。

#### (5) 启动服务

现在可以使用以下命令启动：

```bash
./sparrow startone jupyter
```

### 3、如何进入容器

如果你需要进入运行中的容器进行调试或操作，可以使用 enter 命令。

#### (1) 查看可用容器

如果不指定服务名称，会列出所有正在运行的 sparrow 容器：

```bash
./sparrowtool enter
```

#### (2) 进入指定容器

指定服务名称进入对应的容器：

```bash
./sparrowtool enter {service_name}
```

#### (3) 使用示例

```bash
# 查看所有可用的容器
./sparrowtool enter

# 进入 mysql 容器
./sparrowtool enter mysql

# 进入 redis 容器
./sparrowtool enter redis
```

进入容器后，你将以 root 用户身份登录，工作目录为根目录 `/`，可以自由执行命令进行调试和操作。

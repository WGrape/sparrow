## 2. 安装并启动 Docker / Docker-Compose

> Docker 发布说明：https://docs.docker.com/release-notes/

### (1) 安装

#### a. Linux

如果访问 `github.com` 速度慢或失败，可以尝试切换镜像源。

```bash
# 1. 安装 docker
curl -fsSL https://get.docker.com | bash -s docker --mirror Aliyun

# 2. 使用 curl 安装 docker-compose
# 如果速度很慢，建议使用下面的方式。
curl -L https://github.com/docker/compose/releases/download/1.27.2/docker-compose-`uname -s`-`uname -m` > /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

# 3. 使用 pip3 安装 docker-compose
pip3 install --upgrade pip
pip3 install docker-compose

# 4. 使用 yum 安装 docker-compose
# 注意版本至少为 1.20 或更高，低版本可能导致 compose 失败。
yum install -y docker-compose
```

#### b. Mac

Mac 版 `Docker Desktop` 安装地址：https://docs.docker.com/desktop/release-notes/#4242 （可选择任意版本）。

### (2) 启动

运行 `docker info` 命令检查 Docker 状态，如果未运行，需要先启动 Docker。

<img width="500" src="https://github.com/WGrape/sparrow/assets/35942268/22c72dca-c62a-43d5-81a1-deb4fca749bd" >

```bash
# 检查 Docker 状态
sudo systemctl status docker

# 启动 Docker
sudo systemctl start docker
```

> Compose 失败，执行 compose 时出现了 docker-compose 命令的提示。#5

报错原因是 docker-compose 版本过低，建议升级。[Issues#5](https://github.com/WGrape/sparrow/issues/5)

```bash
# 升级到 1.27.2
curl -L https://github.com/docker/compose/releases/download/1.27.2/docker-compose-`uname -s`-`uname -m` > /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
```

## 4. 架构不匹配：compose 触发 re-pull 并报 "pull access denied"

### (1) 现象

pull 成功、本地已有 `sparrow-app-{service}:latest` tag，但 `docker compose up` 报错：

```
pull access denied for sparrow-app-{service}, repository does not exist or may require 'docker login'
```

### (2) 根本原因

1. `docker pull` 调用时**没有携带** `--platform`，Docker 拉取与宿主机原生架构匹配的镜像（例如 Intel/amd64 机器上拉到 amd64）。
2. 本地 tag `sparrow-app-{service}:latest` 实际是 **amd64** 镜像。
3. `docker compose up` 读取服务定义中的 `platform: linux/arm64`，发现本地镜像架构不匹配。
4. Compose 自动尝试重新拉取镜像以获取正确架构。
5. `sparrow-app-{service}` 不是真实的 registry 仓库名，拉取被拒绝 —— **报错**。

### (3) 解决方法

`docker pull` 必须始终携带 `--platform ${FROM_PLATFORM}`，这在 `dockerhub.sh` 中已强制执行。如果手动执行 `docker pull`，务必显式加上 `--platform`：

```bash
docker pull --platform linux/arm64 lvsid/sparrow-app-mysql:latest
```
